# DAQI Asia 英文店铺前端交付说明

## 本次交付内容

- DAQI Asia 英文版首页
- 独立 Shop 页面：`/shop/`
- 独立分类页：`/product-category/分类slug/`
- Shop 页面展示部分打折、销量/评分较高商品
- 分类页展示对应分类商品
- 关键词搜索展示
- 简单分类筛选和排序
- Cloudflare Pages Functions 服务端代理 WooCommerce API
- 购物车、账户、购买等交易入口跳转原站
- 可上传 Cloudflare Pages 的 DAQI Asia 前端代码

## 不包含内容

- 不恢复 WordPress 数据库
- 不提供 WordPress 后台
- 不做登录、购物车、下单、支付、订单
- 暂不开放站内商品详情页入口，商品交易相关操作跳转原站
- 不做复杂多条件筛选
- 不做人工翻译和商品内容整理
- 不负责部署

## 关于商品数据

当前代码上线到 Cloudflare Pages 后，会优先请求：

```text
/api/products
/api/categories
```

这两个接口在 Cloudflare 服务端读取环境变量：

```text
WC_CONSUMER_KEY
WC_CONSUMER_SECRET
```

因此 API Key 不会写在前端 JS 里，浏览器看不到 Key。

如果 `/api/products` 或 `/api/categories` 请求失败，前端会回退读取：

```text
data/products.json
data/categories.json
```

测试原站 WooCommerce REST API 时，接口没有返回正常 JSON，而是返回了原站 PHP 错误。错误位置显示在原站主题/插件代码中，例如：

```text
wp-content/themes/martfury-child/functions.php:77
```

因此当前交付包内放置了本地目录数据，用于验证 DAQI Asia 页面结构和功能。同步脚本再次运行时，如果原站接口仍报错，会生成 `data/sync-status.json` 记录失败原因，并保留现有 JSON 数据不清空。

客户原站接口修复后，可以使用：

```bash
WC_CONSUMER_KEY="ck_xxx" WC_CONSUMER_SECRET="cs_xxx" node tools/sync-woocommerce.mjs
```

同步真实商品和分类数据。

## 部署方式

将整个 `daqi-static` 文件夹上传到 Cloudflare Pages。

不需要构建命令。

## 本地预览

普通页面预览：

```bash
python3 -m http.server 8098 --bind 127.0.0.1
```

如果要本地预览 `/api/products` 和 `/api/categories` 代理逻辑：

```bash
WC_CONSUMER_KEY="ck_xxx" WC_CONSUMER_SECRET="cs_xxx" node tools/local-preview.mjs
```

然后打开：

```text
http://127.0.0.1:8098
```

注意：如果原站 WooCommerce REST API 仍然返回 PHP 500 错误，本地代理和 Cloudflare 线上代理都会拿不到真实商品数据。
